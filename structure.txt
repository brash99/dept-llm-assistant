Layer			Tunable Parameters

==========================================
Corpus			Inclusion policy, excluded folders, supported parsers

Parsing			OCR on/off, metadata extraction, quality thresholds

Chunking		Chunk size, overlap, max chunks/document, semantic vs fixed chunking

Embeddings		Embedding model, vector dimension, normalization

Vector DB		ANN algorithm, index parameters, distance metric

Retrieval		Top-k, metadata filters, hybrid lexical/vector weighting

Reranking		Cross-encoder model, rerank depth

Prompting		Context window, citation format, system prompt

LLM			Model, temperature, max tokens, reasoning mode

Observations

    • The architecture cleanly separates parsing,
      normalization, retrieval, and reasoning.

    • The normalized Document JSON serves as the
      stable interface between ingestion and AI.

    • Retrieval and reasoning are independent.
      Different embedding models, vector databases,
      rerankers, or LLMs can be substituted without
      changing the earlier stages.

Next Steps

    1. Improve source diversity (document-level
       deduplication).

    2. Add DOCX parser.

    3. Improve citation rendering.

    4. Add corpus statistics and inspection pages
       to the web interface.

    5. Continue evolving the framework toward a
       general institutional knowledge engine.
